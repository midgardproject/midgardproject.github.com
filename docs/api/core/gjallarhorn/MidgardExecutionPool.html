Description

ExecutionPool is a pool of executable objects. Once any executable object has been pushed to the pool, its execute_async method is invoked in separate thread. Internaly, execution pool uses threads pool from GLib. There are no strict rules to define priority or queue of executions. Such are done, once resources are available.

The best approach, is to integrate execution pool with GLib's main loop, because of executors' signals which are not emitted immediately after any execution is done. As a workaround for missed main loop integration, execution pool will force all operations to be executed during pool's destruction phase. Simply, it'll wait for all executions to be done.
